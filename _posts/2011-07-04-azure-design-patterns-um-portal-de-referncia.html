Para iniciar a semana com o pé direito, gostaria de recomendar o portal Azure Design Patternsonde estão catalogados diversos padrões de projeto para a plataforma Windows Azure.

O site foi construído em Silverlight e tem uma experiência de uso muito boa. O título cria uma expectativa imensa então vou logo advertindo: o conteúdo é básico e se destina a descrição dos padrões, não se atendo a parte prática dos mesmos.

Para os que estão estudando o Windows Azure(principalmente os iniciantes), esta é certamente uma ótima ferramenta.

O site se divide em quatro categorias de padrões:

    1. Compute Partterns: Inclui o Windows Azure Compute Service com os Roles (Web Role, Worker Role, etc), bem como o AppFabric Cache Service;
    2. Storage Partterns: Os padrões de armazenamento incluem o Windows Azure Storage Service (Blobs, filas e tabelas) e o Content Delivery Network;
    3. Communication Parttners: Inclui padrões para o Communication Parttners;
    4. Security Patterns: Cobre o Windows Azure AppFabric Access Control Service;
    5. Relational Data Patterns: Isto cobre o SQL Database Service Azure, o SQL Azure Reporting Service, e o Serviço DataMarket (anteriormente chamado Projeto Dallas);
    6. Network Patterns: Aborda padrões com o Windows Azure Connect virtual networking feature.
